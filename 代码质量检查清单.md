# 代码质量检查清单

在提交代码前,请确保完成以下检查项。

## ✅ TypeScript/Vue代码检查

### 类型安全
- [ ] 所有函数参数都有明确的类型定义
- [ ] 所有函数返回值都有明确的类型定义
- [ ] 没有使用`any`类型(除非确实必要)
- [ ] 接口定义完整且准确
- [ ] 运行`npm run build`无类型错误

### 代码质量
- [ ] 没有未使用的变量和导入
- [ ] 没有未使用的函数和方法
- [ ] 没有console.log(除非是必要的日志)
- [ ] 所有TODO注释都有对应的issue
- [ ] 代码格式统一(使用Prettier)

### 错误处理
- [ ] 所有异步操作都有try-catch
- [ ] 所有错误都通过message工具显示
- [ ] 错误日志包含足够的上下文信息
- [ ] 边界条件都有处理

### 性能
- [ ] 没有不必要的响应式对象
- [ ] 频繁触发的事件使用了防抖/节流
- [ ] 大型组件使用了懒加载
- [ ] 没有内存泄漏(事件监听器已清理)

## ✅ Python代码检查

### 文档和注释
- [ ] 所有类都有docstring
- [ ] 所有公共函数都有docstring
- [ ] 复杂逻辑有行内注释
- [ ] docstring包含参数和返回值说明

### 代码质量
- [ ] 没有未使用的导入
- [ ] 没有未使用的变量
- [ ] 命名符合snake_case规范
- [ ] 代码格式符合PEP 8

### 错误处理
- [ ] 所有可能失败的操作都有try-except
- [ ] 异常信息通过log函数输出
- [ ] 关键错误包含traceback
- [ ] 资源使用后正确释放

### 日志
- [ ] 关键操作都有日志记录
- [ ] 日志级别使用正确
- [ ] 日志信息清晰易懂
- [ ] 没有敏感信息泄漏
